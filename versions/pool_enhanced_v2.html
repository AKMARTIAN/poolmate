About the numbers

Molarity for dsDNA: nM = conc(ng/µL) × 10⁶ / (660 × size_bp). Direct nM can be entered when molarity has already been measured. Transfers are calculated as equal fmol per sample, then rounded to the selected increment and audited for post-rounding deviation. Low transfer volumes can be handled by warning only, increasing final pool volume, or producing a pre-dilution plan. In fmol mode, the tool derives target nM as total fmol divided by final volume.

epMotion Equimolar Pooling Helper - User Guide

Quick bench reference for choosing the correct workflow and understanding required fields.

1. Calculate Molarities

Use only the Sample section to convert concentration plus fragment size into nM. This does not calculate pooling volumes.

Minimum: sample concentration + size, or direct nM.

2. Quick Pool

Creates a simple manual pooling table using total fmol or fmol per sample. It does not require platform presets, racks, buffer positions, or epMotion worklists.

Use for: fast bench calculations and manual pooling.

3. Generate Worklists

Full epMotion workflow. Uses sample molarities, final pool target, layout, deck settings, buffer, tips, and exports DNA.csv and Buffer.csv.

Use for: robot execution.

Pooling vs Sample Sections

Sample section: what you currently have, namely sample names, wells, concentrations, sizes, and direct nM values.

Create pool parameters: what final pool you want, namely target nM or fmol, final volume, and layout.

Plate Map

The live plate map mirrors the source wells entered in the sample table. Sample names are shown inside the selected wells. Use it to confirm rack loading before running the epMotion.

Output Files

DNA.csv: DNA transfer worklist. Buffer.csv: buffer transfer worklist. Worksheet: calculation record. Inputs: saved run settings.
